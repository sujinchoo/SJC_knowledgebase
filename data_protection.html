Storage & Data Protection

Snapshot · Replication · Backup · Disaster Recovery (DR) 개념 요약

1️⃣ 기본 개념

Snapshot특정 시점의 데이터를 빠르게 복사한 가상 이미지. 파일 삭제/오류 발생 시 빠른 복구 가능.
Replication데이터를 다른 장비나 사이트로 복제하여 장애 발생 시 서비스 연속성을 보장.
Backup정기적으로 데이터를 별도의 매체(디스크, 테이프, 클라우드)에 저장해 장기 보존.
Disaster Recovery (DR)재해 상황에서도 업무 연속성을 유지하기 위한 복구 체계. 보통 원격지 복제(Replication)와 DR 테스트를 포함.

2️⃣ RPO / RTO 정의

RPO (Recovery Point Objective)“얼마나 최근의 데이터까지 복구할 수 있나?” — 백업 주기나 복제 주기에 따라 달라짐.
RTO (Recovery Time Objective)“서비스를 얼마만에 다시 정상화할 수 있나?” — 복구 절차의 속도와 인프라 설계에 따라 달라짐.
💡 예시: RPO=4시간, RTO=2시간이라면 장애 발생 시 4시간 이전의 데이터까지 복구하고, 전체 서비스를 2시간 안에 복원해야 함.

3️⃣ 백업 전략 종류

Full Backup전체 데이터를 매번 백업. 복구는 빠르지만 시간·공간 많이 소모.
Incremental Backup이전 백업 이후 변경된 부분만 백업. 공간 효율 좋지만 복구 시 여러 버전 필요.
Differential Backup가장 최근 Full 이후 변경분만 백업. Full + Differential 두 세트만 있으면 복구 가능.

4️⃣ 데이터 보호 아키텍처

Local Snapshot동일 스토리지 내에서 빠른 복구를 위한 기능 (NetApp, Lenovo DM/DE Series 지원)
Remote ReplicationDR 사이트로 데이터 실시간 복제 (Async 또는 Sync)
Cloud Backup오프사이트 보관용. Ransomware 대비 및 장기보존 목적
🧩 TAM 관점: 고객의 RPO/RTO 목표를 듣고 백업·복제 구조를 추천할 수 있어야 함.

5️⃣ TAM 대응 체크리스트

1. 백업 성공률최근 백업 로그 7일간 성공률 확인 (95% 이상 목표)
2. Snapshot 관리보존 기간, 자동 삭제 정책 점검
3. DR 테스트연 1회 이상 실제 복구 시나리오 테스트 권장
4. 복제 지연 (Lag)Replication delay 지표 모니터링 → 네트워크 병목 파악
5. 데이터 무결성Checksum 또는 Hash 검증으로 손상 여부 점검
