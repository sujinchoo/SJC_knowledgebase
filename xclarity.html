Lenovo XClarity

Lenovo 인프라를 중앙에서 자동 발견·모니터링·패치·통합하는 경량 시스템 관리 플랫폼

1️⃣ XClarity 개요

Lenovo XClarity는 서버, 네트워크, 스토리지를 에이전트 없이(agent-free) 중앙에서 관리하도록 만든 관리 플랫폼입니다. 데이터센터에 있는 Lenovo 장비를 자동으로 발견하고, 인벤토리를 수집해 한눈에 상태를 볼 수 있게 해줍니다.

💡 관리자 입장에서는 “장비마다 붙어서 보는 게 아니라, 한 대시보드에서 전부 보는 것”이 핵심입니다.

2️⃣ 구성 요소

XClarity Administrator 가상 어플라이언스로 배포하는 중앙 관리 서버. 장비 자동 발견, 인벤토리, 이벤트/알람, 펌웨어 일괄 업데이트, OS 배포까지 담당.
XClarity Controller (XCC) ThinkSystem 서버에 내장된 BMC/관리 엔진. 전원 제어, 원격 콘솔, 하드웨어 상태 수집을 표준화해 Administrator에서 쉽게 모니터링하도록 함.
XClarity Integrator vCenter, Microsoft, ServiceNow 등 기존 ITSM/가상화 콘솔에 Lenovo 정보를 끌어와 한 화면에서 운영하도록 하는 플러그인 계열.
🧩 TAM 관점에서는 “고객 인프라에 XClarity Administrator가 설치되어 있나?”를 먼저 확인하고, 없으면 PoC부터 제안하는 흐름이 좋습니다.

3️⃣ 주요 기능

자동 발견 & 인벤토리 네트워크상 Lenovo 장비를 스캔해 자동으로 등록하고, 모델/시리얼/펌웨어 버전/상태를 한 화면에 표시.
이벤트·알람 통합 서버나 스위치에서 발생한 경고를 중앙으로 모아 대시보드에서 확인. 심각도별로 필터링 가능.
펌웨어/드라이버 일괄 배포 지원 매트릭스에 맞춰 여러 대를 한 번에 업데이트해서 “장비마다 버전이 다른” 문제를 줄임.
OS·구성 패턴 배포 동일한 서버를 여러 대 구축할 때 패턴을 만들어 빠르게 배포. 신규 팟(pod) 오픈 시 유용.
REST API 제공 CI/CD나 사내 포털과 연계해서 자동화 가능. Integrator가 이것을 활용해 타 시스템과 연결.

4️⃣ TAM 관점 활용 포인트

  • 헬스체크 리포트 소스: XClarity에서 뽑은 인벤토리·이벤트 리스트를 월간 리포트 자료로 사용
  • 펌웨어 미준수 장비 식별: 고객 장비 중 FW가 뒤처진 것만 필터링해 변경 요청(Change)으로 연결
  • 콜홈(Call Home) 연계 여부 확인: Pro 라이선스나 지원 옵션 사용 시 장애 티켓 자동 생성되는지 확인
  • 다른 콘솔과의 중복 제거: vCenter/Prism만 보던 고객에게 “HW는 XClarity로”라고 역할을 나눠주면 운영이 단순해짐
💡 면접이나 고객 미팅에서 “XClarity 있으면 펌웨어 불일치·인벤토리 수집·알람 통합이 쉬워집니다” 한 줄만 넣어도 전문성 어필됩니다.

5️⃣ 도입 / 운영 체크리스트

1. 지원 대상 장비 확인 고객이 쓰는 서버/스위치/스토리지가 XClarity Administrator에서 관리 가능한 모델인지 확인.
2. 네트워크/방화벽 관리망에서 장비 IP에 접속 가능한지, 인터넷을 통해 펌웨어/라이선스 서버에 접근 가능한지 점검.
3. 계정/권한 장비에 접속할 관리자 계정을 XClarity에 등록해두어야 자동 수집이 제대로 동작.
4. 리포트 주기 월 1회 인벤토리 내보내기 또는 이벤트 요약을 표준화해서 TAM 리포트에 첨부.
5. Integrator 필요 여부 고객이 이미 ServiceNow, vCenter를 쓰고 있다면 해당 통합 모듈까지 제안.
